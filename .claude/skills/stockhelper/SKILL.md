# StockHelper — 智能股票数据分析助手

> 面向个人投资者的轻量级智能股票数据分析工具。
> 核心能力：**产业链按需调研** | **业绩爆发增长筛选** | **技术分析与报告生成**

---

## 架构设计

采用 **渐进式加载架构**（参考 Buffett Skills 设计模式）：

```
SKILL.md（核心逻辑，始终在内存）
  ├── references/data-sources.md      （数据源参考，按需加载）
  ├── references/analysis-models.md    （分析模型详情，按需加载）
  ├── references/industry-chains.md    （产业链基线数据，按需加载）
  └── references/report-templates.md   （报告模板，按需加载）
```

**三条执行路径**：

| 路径 | 适用场景 | Token 成本 | 加载文件 |
|------|---------|-----------|---------|
| **Path A — 快速查询** | 行情、自选股管理、简单技术指标 | 低 | 仅 SKILL.md |
| **Path B — 产业链调研** | 按需产业链分析、增长筛选 | 中 | SKILL.md + references |
| **Path C — 深度分析报告** | 完整基本面+技术面报告 | 高 | SKILL.md + 所有 references |

## 业绩爆发增长筛选模型（FR-08）

七维因子评分体系：

| 维度 | 权重 | 指标 | 满分条件 |
|------|------|------|---------|
| 营收增长 | 25% | 营收同比增长率 | ≥ 50% |
| 利润增长 | 25% | 归母净利润同比增长 | ≥ 100% |
| 盈利能力 | 15% | ROE | ≥ 15% |
| 增长质量 | 10% | 经营性现金流/净利润 | ≥ 0.8 |
| 盈利趋势 | 10% | 毛利率变化 | 同比提升 |
| 估值匹配 | 10% | PEG | < 1.0 |
| 季度加速 | 5% | 环比增速变化 | 连续加速 |

**特殊标识**：
- 🔥 **业绩爆发**：营收增速 > 50% 且 净利润增速 > 100%
- 📈 **高增长**：营收增速 > 30% 且 净利润增速 > 50%
- ⚡ **困境反转**：过去两年利润为负，当期转正且增速 > 100%
- 🚀 **增速加速**：连续两季度增速环比提升

## 产业链瓶颈分析（Serenity 方法论）

**核心逻辑**：确认大趋势 → 反推供应链 → 识别瓶颈环节 → 验证证据 → 评估催化剂

**瓶颈评分卡（1-10 分）**：

| 评分因子 | 权重 | 说明 |
|---------|------|------|
| 公司数量 | 40% | 该环节上市公司数量越少越瓶颈 |
| 技术壁垒 | 30% | 环节涉及的核心技术难度越高越瓶颈 |
| 扩产难度 | 20% | 产能建设周期和资本开支要求 |
| 客户锁定 | 10% | 认证周期长、替换成本高加分 |

**评分解读**：
- 🟢 **1-3 分**：竞争充分，不太可能出现供给瓶颈
- 🟡 **4-6 分**：有一定稀缺性，需要关注变化
- 🔴 **7-10 分**：高瓶颈环节，若需求爆发弹性最大

## 质量控制（HARD-GATE）

参考 UZI-Skill 的事实核查机制：

### 必须遵守的规则

1. **严禁编造产业链关联** — 如果某公司与目标产业没有公开的业务关联，不得强行关联
2. **数据必须有来源** — 所有分析结论须标注数据来源或注明"待验证"
3. **不确定必须标注** — 使用"据公开报道""可能需要进一步验证"等措辞
4. **交叉验证原则** — 单一数据源的结论需标注"仅参考单一来源"

### 自查清单

- [ ] 产业链各环节公司是否有公开业务依据？
- [ ] 财务数据是否标注了报告期？
- [ ] 增长判断是否基于真实财报数据？
- [ ] 瓶颈评分是否有合理的逻辑支撑？

## CLI 命令参考

### 自选股管理
```
stock list                   查看自选股列表
stock add <code>             添加自选股（如 600519）
stock remove <code>          删除自选股
```

### 个股分析
```
stock detail <code>          查看个股详情与技术指标
stock report <code>          生成 HTML 分析报告
```

### 综合看板
```
stock dashboard              查看自选股综合看板
stock screener               全市场业绩爆发增长扫描
```

### 产业链分析（核心功能 🔥）
```
chain research <industry>    调研指定产业链并生成全景图谱（核心）
chain view <industry>        查看已缓存产业链详情
chain list                   查看已调研过的产业链列表
chain refresh <industry>     重新调研更新产业链数据
chain compare <industry>     对比产业链各环节公司财务数据
chain growth <industry>      产业链内业绩爆发增长筛选
chain boom <industry>        产业链景气度热力图
```

### 使用示例
```
chain research 低空经济         调研低空经济产业链
chain research 人形机器人        调研人形机器人产业链（无需预制）
chain compare 半导体            对比半导体各环节公司财务
chain growth 人工智能           筛选AI产业链高增长标的
stock screener                 全市场扫描业绩爆发公司
```
